# 📝 大语言模型 (LLM) 基础知识体系笔记

## 一、 LLM 的核心运作机制
* **文字接龙 (Next-Token Prediction)**：大模型不“理解”文字，而是基于海量数据计算概率，预测下一个最可能出现的词。
* **词元 (Token)**：AI 处理文本的基本单位。
  * 英文 1 Token ≈ 0.75 个单词；中文 1 Token ≈ 0.5 ~ 1 个汉字。
  * **测试注意**：API 的计费规则和输入输出上限，全部以 Token 为单位衡量。

## 二、 模型的“记忆”与“性格”
* **上下文窗口 (Context Window)**：模型的“短期记忆容量”（单次处理 Token 的最大数量）。超过此限制会导致遗忘或报错。
* **温度值 (Temperature)**：控制输出的创造力与随机性。
  * `0.0`：高度确定、严谨（**自动化测试强制要求设为 0**）。
  * `> 0.8`：发散性强、每次输出可能不同。
* **无状态 (Stateless)**：API 本身没有记忆。多轮对话必须在代码中手动维护历史消息列表（Context），并在每次请求时全量发送。

## 三、 API 调用的角色协议 (Roles)
* **System (系统)**：拥有最高权重。用于设定 AI 的全局人设、语气和底线规则（如：“你是一个严格的资深测试开发，只输出合法的 JSON”）。
* **User (用户)**：人类发送的具体指令/当前任务。
* **Assistant (助手)**：AI 历史生成的回复（用于拼接多轮对话上下文）。

## 四、 模型的致命缺陷 (自动化测试监控重点)
* **幻觉 (Hallucination)**：当模型不知道答案时，会基于概率“一本正经地胡说八道”。这是 AI 测试最大的 P0 级 Bug。
* **知识截止 (Knowledge Cutoff)**：模型不知道训练数据截止日期之后发生的新事件。
* **缺乏推理 (Lack of Reasoning)**：面对复杂数学或多层级业务逻辑极易出错，需要通过 Prompt（如“一步步思考”）或外部代码辅助。

## 五、 高阶生态体系 (框架进阶方向)
* **Embeddings (词向量)**：将文本转化为多维数组（坐标），用于计算文本间的语义相似度。
* **RAG (检索增强生成)**：给 AI 挂载外部知识库。流程：先检索本地资料（如公司接口文档） -> 拼接进 Prompt -> 交给 AI 总结回答。（**解决“幻觉”的核心方案**）
* **Agents (智能体)**：赋予 AI 调用外部工具（如执行 Python 代码、查数据库）的能力。
